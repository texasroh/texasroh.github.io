블로그를 다시 시작해야겠다고 마음먹고 옛 Google Blogger를 열어봤다. 학부 시절 끄적인 글들이 그대로 남아 있었는데, 톤·내용·디자인 모두 그 시점의 흔적이라 지금 정체성과 맞지 않았다. **옮기지 않기로 했다.** 옛 Blogger는 그대로 두고, 새 챕터로 시작.

이 글은 그 새 시작을 어떤 플랫폼 위에서 할지 정하는 의사결정 과정이다.

## 후보 정리

처음에 따져본 옵션:

- **Blogger 잔류** — 이미 거절. 디자인·기능 제어가 너무 제한적이고, Google Blogger 자체가 방치 상태에 가깝다
- **Jekyll** — GitHub Pages 공식 지원, 마크다운 기반의 표준 블로그 컨벤션
- **Next.js static export** — React/TS 친숙, MDX 지원, 정적 사이트로 GitHub Pages 호스팅 가능
- **Astro** — Jekyll급 성능 + React 컴포넌트 혼용. 매력적이지만 새 도구 학습 부담

## 첫 시도: Jekyll (며칠 만에 떠남)

처음엔 Jekyll로 시작했다. 컨벤션이 표준화돼 있어 빠르게 띄울 수 있었지만, 며칠 만에 마찰이 쌓였다.

- **Liquid 템플릿의 한계** — 컬렉션·필터링이 복잡해지면 디버깅이 고통스럽다. JSX/TS 환경이 이미 익숙하니 더 그렇다
- **Ruby/Bundler 환경** — 평소 Node 외엔 안 쓰니 toolchain이 한 가지 더 늘어나는 게 거슬렸다

며칠 만에 결심했다. **이건 안 되겠다.**

## 정착: Next.js static export

다음 후보로 Next.js를 골랐다. 결정 근거:

- **타입 안전성** — 콘텐츠 로더, i18n 사전, 데이터 스키마를 TS로 관리. 빌드 타임에 슬러그 중복 같은 실수도 잡힌다
- **MDX** — 본문 안에 React 컴포넌트를 그대로 박을 수 있다. `Callout` 같은 박스를 직접 만들어 쓰는 게 자연스럽다
- **개발 환경 일관성** — 평소 쓰는 Node 도구로 끝
- **GitHub Pages는 정적 호스팅에만** — `output: 'export'`로 빌드한 `out/`을 GitHub Actions가 업로드하는 구조. 서버 없음

Astro도 매력적이었는데, 한 가지 더 도구 익히는 비용 대비 차별화 이점이 작아 보였다. **익숙한 무기로 가자**가 결론.

## 사이트 구조

```
app/
  (site)/[lang]/
    blog/
      page.tsx              # 글 목록
      [slug]/page.tsx       # 글 상세
components/                  # 공용 UI
lib/                         # 콘텐츠 로더, i18n
_posts/{ko,en}/             # 블로그 글 (Markdown / MDX)
public/assets/              # 정적 자원
```

글은 마크다운으로 둔다. 글쓰기·편집은 에디터에서, 빌드·배포는 자동으로.

## URL 정책

블로그 URL은 슬러그만 쓴다.

```
/ko/blog/from-blogger-to-nextjs/
```

연·월·일은 빼고, **같은 언어 안에서 슬러그가 겹치면 빌드가 실패하도록** 막아뒀다. 두 글이 한 URL을 두고 충돌하는 상황은 사전에 차단.

## 다국어 정책

`/ko/...`, `/en/...` 두 트리. 각 글은 같은 슬러그로 매칭된다. 헤더의 언어 토글이 같은 슬러그를 찾아 이동하고, 번역본이 없으면 해당 언어의 블로그 목록으로 떨어진다.

번역은 의무 아님. **글마다 자연스러운 언어**로 쓰고, 글로벌 청중에게 닿을 만한 글만 영문판도 작성한다. 한국어가 기본, 영어는 골라서.

## 앞으로

옛 Blogger의 글들은 옮기지 않는다. 새 챕터에서 쓸 만한 글감:

- **의사결정 회고** — 왜 X 대신 Y를 골랐나
- **인시던트·디버깅** — 프로덕션에서 잡은 버그 이야기
- **실무 적용기** — "써봤다"가 아니라 "맥락에서 어떻게 적용됐나"

빠른 페이스보단 신중한 페이스로. 첫 방문자가 무작위로 누른 글의 인상이 곧 블로그 인상이다.
